Theory behind the steps

🔍

Generating symbols

A symbol is generating if it can derive at least one string of terminal characters. Found using fixed-point iteration — start with terminals, spread to non-terminals whose every rule resolves.

🌐

Reachable symbols

A symbol is reachable if it can be reached from the start symbol S by following production rules. Found by BFS/DFS from S through all rules. Unreachable symbols are dead code.

∅

Nullable symbols

A symbol A is nullable if A ⇒* ε. Found by checking for direct ε rules, then propagating: if all symbols in a rule are nullable, the LHS is too.

⛓️

Unit pairs

A unit pair (A, B) means A ⇒* B through only unit productions. Computing the transitive closure lets us replace all chains A → B → C → d with A → d directly.

✓

Why simplify?

Simplified grammars are required for most parsing algorithms (CYK, Earley). Removing useless symbols reduces state space. Removing ε and unit rules makes analysis easier.

≡

Language equivalence

Every step preserves the language L(G). The simplified grammar generates exactly the same strings as the original — except possibly ε if S was nullable and not the start.
